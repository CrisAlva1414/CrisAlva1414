# Hey, soy Cristian 👋

Indie dev de Santiago, Chile. Lo que empezó como hobby terminó siendo trabajo — y no me arrepiento.

Me muevo en el espacio entre infraestructura, automatización e inteligencia artificial. Puedo tomar una idea, construir la herramienta, montarla en producción y dejar corriendo monitoreo y CI/CD. Sin depender de terceros para la parte difícil.

Mi sesgo fuerte es hacia el **open source**, la **gobernanza de datos propia** y los sistemas que funcionan bien sin que nadie tenga que mirarlos.

## Lo que construyo

**Sistemas de IA aplicada** — pipelines RAG, chatbots con retrieval real, diagnóstico y mejora continua de corpus vectoriales. No demos: cosas que corren en producción respondiendo clientes reales.

**Infraestructura open source para empresas** — stacks completos basados en software libre con énfasis en autonomía, rendimiento y resiliencia. Sin lock-in. Sin pagar por lo que puedes operar tú mismo.

**Automatización de procesos** — workflows con n8n, integraciones entre APIs, bots de WhatsApp, alertas, reportes automáticos. El tipo de cosas que antes hacía alguien manualmente.

**Herramientas de datos y BI** — desde ingesta y transformación hasta dashboards. Experiencia con la suite Apache (Superset, Kafka, Airflow, Spark) y pipelines analíticos sobre datos reales.

**Hardware + software** — proyectos con ESP32, sensores, domótica, impresoras térmicas con servidor ESC/POS propio, y sistemas embebidos corriendo en SBCs ARM.

## Proyectos destacados

| Proyecto | Descripción |
|---|---|
| **Ruki-System** | Servidor doméstico sobre OrangePi 5 (ARM64). Docker, Cloudflare Tunnel, n8n, monitoreo. Sin puertos abiertos al exterior. |
| **RAG Diagnostics** | Sistema de diagnóstico, mantenimiento y testing competitivo de corpus RAG. Pipeline completo: auditoría → reconstrucción LLM → evaluación con juez → upsert en Qdrant. |
| **ENDDEIE 2023** | Pipeline analítico completo sobre datos educacionales reales. 1.174 establecimientos, clustering KMeans, PCA, UMAP, Random Forest, bootstrap. |
| **ERP Q-Cube Demo** | Sistema ERP de demostración para gestión logística con impresión térmica. React 19 + TypeScript + Vite. |
| **ESC/POS Server** | Servidor de impresión sin CUPS ni drivers. FastAPI, dithering Floyd-Steinberg, cola persistente, API HTTP completa. Corre en Raspberry Pi. |
